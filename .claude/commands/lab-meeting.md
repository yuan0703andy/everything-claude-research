# /lab-meeting

## 目的
模擬每週 Lab Meeting，審查進度、討論假說、更新排名。

這是研究團隊的定期同步機制，確保所有人對專案狀態、假說優先級、資源分配有共識。

## 使用時機
- 建議每週固定時間執行（例如每週一早上）
- 重大專案里程碑時
- 需要團隊集體決策時
- 有新假說需要審查時

## 前置準備
Lab Manager 應該在會議前準備：
1. 上週的會議記錄和行動項目
2. 各專案的最新狀態
3. 假說排名表
4. 待決事項清單

## 標準議程

### 1. 開場與行動項目回顧 (5 分鐘)
**主持**: Lab Manager

**內容**:
- 回顧上週會議的決議
- 檢查行動項目完成情況
- 未完成項目的障礙識別

### 2. 專案狀態報告 (5-10 分鐘)
**主持**: Lab Manager
**參與**: 所有成員

**內容**:
- 每個活躍專案的狀態更新
- 進度百分比
- 本週完成的里程碑
- 遇到的問題和風險
- 資源使用情況

**格式**:
```
專案 A:
- 狀態: 🟢/🟡/🔴
- 進度: XX%
- 本週: [完成了什麼]
- 下週: [計畫做什麼]
- 風險: [有什麼障礙]
```

### 3. 假說審查 (20-30 分鐘)
**主持**: 依假說類型選擇合適的 Senior Postdoc

對新假說或有重大更新的假說進行審查：

#### 3.1 假說報告
**Theorist** 報告：
- 假說的理論動機
- 核心主張和機制
- 與現有文獻的關係
- 為什麼這個假說重要

#### 3.2 可行性評估
**Experimentalist** 評估：
- 驗證方案
- 資源需求（數據、時間、計算）
- 技術風險
- 預期時間線

#### 3.3 方法論意見
**Methodologist** 提供：
- 方法論適當性
- 潛在偏誤和陷阱
- 是否符合領域標準
- 預期審稿人關注點

#### 3.4 討論與決議
**全體** 討論：
- 優勢和劣勢
- 與其他假說的比較
- 優先級判斷
- 資源分配

**決議**:
- ✅ 推進：分配資源開始驗證
- 🔄 修改：需要調整後再審查
- ⏸️ 擱置：暫時不推進，但保留
- ❌ 放棄：不再考慮

### 4. 假說排名更新 (10 分鐘)
**主持**: Lab Manager
**方法**: Elo 系統成對比較

如果有多個假說需要排名：
1. Lab Manager 提出成對比較
2. 團隊根據多個維度投票：
   - 理論重要性
   - 可行性
   - 新穎性
   - 影響力
3. 更新 Elo 分數
4. 記錄排名變化

### 5. 問題討論 (10-15 分鐘)
**主持**: PI (或最相關的 Senior Postdoc)

討論需要集體智慧的問題：
- 卡住的技術問題
- 方向性疑問
- 資源衝突
- 跨專案協調

### 6. 資源與時間線 (5 分鐘)
**主持**: Lab Manager

- 資源分配檢視
- 即將到來的截止日期
- 下週重點任務
- 需要外部協助的事項

### 7. 下週計畫 (5 分鐘)
**主持**: Lab Manager

每位成員簡要說明下週目標：
- Theorist: [下週任務]
- Experimentalist: [下週任務]
- Methodologist: [下週任務]

### 8. 結束與記錄 (5 分鐘)
**主持**: Lab Manager

- 總結決議事項
- 確認行動項目和責任人
- 設定下次會議時間

## 最佳實踐

### 會議效率
- 控制時間，嚴格按議程進行
- 技術細節討論可以會後單獨進行
- 如果討論超時，標記為待辦事項

### 決策品質
- 所有決議都要有明確的行動項目和責任人
- 不確定時承認不確定，避免倉促決策
- 記錄不同意見，不強求共識

### 團隊協作
- 輪流讓不同人主持不同議題
- 鼓勵建設性批評
- 慶祝進展和里程碑

### 持續改進
- 每月回顧會議流程的有效性
- 根據專案需求調整議程
- 保持靈活性

## 特殊情況

### 緊急會議
如果有緊急問題需要即時討論：
```bash
/lab-meeting --emergency "問題描述"
```
只處理該緊急問題，使用簡化議程。

### 專案啟動會議
新專案開始時：
```bash
/lab-meeting --project-kickoff "[專案名]"
```
專注於專案目標、資源分配、時間線設定。

### 專案結案會議
專案結束時：
```bash
/lab-meeting --project-closure "[專案名]"
```
回顧成果、總結經驗、更新知識庫。

## 注意事項
- Lab Meeting 不是研討會，不要深入技術討論
- 重點是同步資訊、做出決策、分配任務
- 如果連續幾週沒有實質進展，需要反思是否有系統性問題
- 會議記錄要及時整理，最好會後 24 小時內完成
